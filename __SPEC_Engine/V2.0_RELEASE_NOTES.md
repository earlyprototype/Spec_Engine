## What is v2.0?

Constitutional Slow-Code Engine v2.0 merges SPEC Engine's structural rigour with Lia Workflow Specs' troubleshooting philosophy, creating a unified framework that prioritises both execution reliability AND human understanding.

**Philosophy:** Understanding over speed, when understanding matters.

## Headline Features

### üîç Troubleshooting Ecosystem
**The "Troubleshooting Cliff" Solution**

Three specialized TOOLSPECs for systematic debugging:

| TOOLSPEC | Purpose | When to Use | Time |
|----------|---------|-------------|------|

### üéì Education Mode
**Learning-Focused Execution**

New execution mode alongside Silent, Collaborative, Dynamic:

**Education Mode Behaviour:**
1. Runs autonomously most of the time
2. Pauses at critical steps/decisions
3. **EXPLAINS** situation and available approaches
4. **PRESENTS** alternatives with pros/cons
5. **RECOMMENDS** approach with rationale
6. **REQUESTS** human review
7. **EXECUTES** approved approach
8. **EXPLAINS** outcome and lessons learned

**Configuration:**
```toml
[execution]
default_mode = "education"

[education]
enabled = true
approval_gates = "increased"       # How many checkpoints
rationale_required = true          # Always explain why
alternatives_shown = true          # Show options
comparison_depth = "detailed"      # Depth of analysis
learning_pace = "moderate"         # Frequency of pauses
```

**Use Cases:**
- Learning new frameworks or patterns
- Training on project-specific workflows
- Building understanding before autonomous execution
- Onboarding team members to codebase

**Impact:** "AI teaches you while you work" - bridges the understanding gap that creates technical debt.

## Frequently Asked Questions

**Q: Do I need to upgrade existing SPECs?**  
A: No. All v1.x SPECs remain fully functional.

**Q: What if something breaks?**  
A: Unlikely (100% backward compatible). If issues occur, see MIGRATION_v1_to_v2.md rollback section.

**Q: How do I use new TOOLSPECs?**  
A: Navigate to `_TOOLSPECs/[NAME]/`, open the spec file, follow prompts. See GETTING_STARTED_v2.md for walkthrough.

**Q: What happened to MCP server?**  
A: Deferred to v2.1 (Q1 2026). Not critical for v2.0 core features.

**Q: Where are the other 13 TOOLSPECs?**  
A: Phased rollout. Priority was troubleshooting ecosystem (complete). Remainder in v2.1-2.3.

**Q: Can I contribute TOOLSPECs?**  
A: Yes! Follow constitutional guidelines, test thoroughly, document well. See README_WORKFLOW_LIBRARY.md for requirements.

**Q: How do I know if notepad is working?**  
A: After SPEC execution, check for `notepad_[descriptor].md` in the SPEC directory. If using v1.x SPEC, notepad won't generate (upgrade required).

**Q: Is education mode worth using?**  
A: Yes if learning. No if experienced with the workflow. Balance understanding (education) vs speed (dynamic/silent).
